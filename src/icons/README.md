# AegisGate Lens Icons

The manifest references four PNG icons:

- `icon-16.png` (16x16)
- `icon-32.png` (32x32)
- `icon-48.png` (48x48)
- `icon-128.png` (128x128)

These PNGs are **generated by the build tool** at `tools/build-lens-extension/` in the Platform monorepo, from a single SVG source in `tools/build-lens-extension/assets/lens-icon.svg`. The SVG is hand-crafted in the AegisGate visual style: a shield with a "🛡️" motif on a blue background.

**The PNGs are NOT committed to this repo** (per the constraint: no binary assets unless strictly necessary; the build tool generates them).

**For local development without the build tool:** create a 1x1 transparent PNG and place it at each of the four paths. The Chrome Web Store will reject the extension if the icons are missing or wrong size; this is a v0.2+ concern (the build tool will be the source of truth).

**The build tool's icon generation step** (in `tools/build-lens-extension/assets/build_icons.go`):
1. Reads the SVG.
2. Rasterizes it to PNG at 16, 32, 48, 128 px.
3. Writes the four files to the extension's `dist/icons/` directory.
4. Logs the SHA-256 of each file as part of the build's `INVENTORY.txt`.

The implementation uses Go's `image/png` package for the output and a hand-rolled SVG-to-bitmap converter (no third-party libraries; see `tools/build-lens-extension/assets/svg_to_png.go`).
